# Contributing

We welcome contributions and are really glad about your involvement in the project.

## Reporting Feedback

Use GitHub issues to report a problem or request a new feature or other improvements.

## Contributing Code

Code contributions are highly appreciated. Bug fixes and small enhancements can be handed in as pull-requests. For larger changes and features, please open a GitHub issue with your proposal first so the solution can be discussed.

### Commit Guidelines

All your commits must follow the [conventional commit message format](https://www.conventionalcommits.org/en/v1.0.0/#summary).

The commit message should be prefixed as follows (examples):

- `fix:` a commit of the type `fix` patches a bug in your codebase (this correlates with ''PATCH'' in Semantic Versioning): 1.0.0(+1)
- `feat:` a commit of the type `feat` introduces a new feature to the codebase (this correlates with ''MINOR'' in Semantic Versioning): 1.0(+1).0
- `feat!:` this type of commit describes a new feature that contains a breaking change (this correlates with ''MAJOR'' in Semantic Versioning): 2(+1).0.0
- `chore:` a minor change, unrelated to the code's execution (e.g. documentation, readme, pipeline changes, etc)

Before creating a pull request, please verify the code can be executed successfully in your local environment.
